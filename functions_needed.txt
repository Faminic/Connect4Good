[DONE] /login: Logs in a user by verifying their email and password combination.

[DONE] /user/delete_user: Deletes the user

[DONE] /user/is_admin: Checks if user is admin




-------[ONLY ALLOW/SHOW THE FOLLOWING FUNCTIONALITIES TO ADMIN USERS]-------

[DONE] /user/promote_admin: Allows admin to Grant admin privileges to a user, allowing them to perform administrative tasks within the application.

[DONE] /user/demote_admin: Allows admin to remove admin privileges from a user, restricting their access to administrative tasks.

[DONE] /event/create_event: Enables the admin to create a new volunteer event, providing details such as title, dates, location, description, and tasks to populate the event in the database.

[DONE] /event/update_event: Modifies details of an existing event such as title, dates, location, description, or tasks.

[DONE] /event/delete_event: Allows the admin to delete a specific event from the database, removing all associated information such as event details, tasks, and registered users.

[DONE] /event/get_users_registered: Retrieves a list of all students (users) registered for a specific event

[DONE] /admin/get_user: Allows the admin to view the profile of a specific student (user), providing access to their details such as phone number, country, city, interests, and past volunteer experience.

[DONE] /admin/kick_user: Removes a student (user) from a specific event, revoking their registration and removing their ID from the list of users registered for that event.
